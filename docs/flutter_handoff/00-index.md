# Handoff To Flutter Developer

هذا المجلد مخصص لتسليم التعديلات الأخيرة إلى مطور Flutter بشكل واضح وسريع.

## الملفات

- راجع: `01-auth-and-security.md`
- يشرح تغيير كلمات المرور، الجلسات، والتوكن.

### 2. هيكل الردود الموحّد
- راجع: `02-api-response-and-token.md`
- يشرح شكل كل رد وكيفية إرسال التوكن.

### 3. العقود الجديدة للـ API
- راجع: `03-endpoints-and-request-bodies.md`
- يشرح ما الذي يجب أن يرسله تطبيق Flutter لكل endpoint بعد التعديلات.

### 4. تحسينات الواجهة داخل لوحة التحكم
- راجع: `04-dashboard-ui-refresh.md`
- هذا الملف ليس أساسيًا لمطور Flutter، لكنه يشرح التغييرات البصرية التي صارت في واجهة الويب.

### 5. ملاحظات التشغيل والمهاجرات
- راجع: `05-backend-settings-and-migrations.md`
- يشرح `.env` والـ migrations المطلوبة بعد سحب التعديلات.

### 6. النشاطات والدروس
- راجع: `06-activities-and-lessons-handoff.md`
- يشرح endpoints الجديدة، الفلاتر، وبنية البيانات الخاصة بالنشاطات والدروس.

### 7. نظام النقاط والميزات الجديدة
- راجع: `08-points-system-handoff.md`
- يشرح نظام النقاط، الاختبارات، الملاحظات، سلوكيات الطلاب، وسلوك حسن.

### 8. تحديث Pages API وبيانات الأجزاء والسور
- راجع: `09-pages-api-updates-handoff.md`
- يشرح التحديثات الجديدة للـ pages API، بيانات الأجزاء (Juz) والسور (Surah)، والفلترة.

## الأهم لمطور Flutter

1. تسجيل الدخول الآن يرجع `token`.
2. أي endpoint محمي يحتاج `Authorization: Bearer <token>`.
3. الردود أصبحت موحدة.
4. أغلب المسارات الخاصة بالمشرف لم تعد تحتاج `teacher_id`.
5. بعض الطلبات التي كانت تعتمد على الأسماء أصبحت تعتمد على `IDs`.
6. النشاطات والدروس أصبح لها عقود رسمية بدل الـ placeholders القديمة.

## ملاحظات سريعة

- endpoint النسخة `GET /api/version` ما زال public.
- إذا كان تطبيق Flutter يستخدم `admin_name` أو `student name` في بعض الطلبات القديمة، يجب تعديلها حسب الملفات التالية.
